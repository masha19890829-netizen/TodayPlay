# TodayPlay V0.9.53 感性化布局重构计划书 - 2026-06-12

## 1. 问题判断

当前 V0.9.52 已经解决了同城路线、频道和商店截图路径，但用户打开后的情绪仍然偏弱：

- 首页虽然有图，但第一眼仍像信息列表，缺少“今晚一起出去”的情绪入口。
- 路线卡同时展示标题、标签、证明、站点、原因和三个按钮，信息密度偏高。
- 动效存在于部分组件，但首屏没有明显的动态氛围。
- 恋爱感主要靠文字表达，缺少图片、光线、节奏和情绪按钮来表达。

本轮定义为 P1 首次体验与视觉转化问题：先解决“干巴巴、文字多、没感觉”，再继续扩功能。

## 2. 团队分工结论

| 工种 | 本轮判断 |
| --- | --- |
| 首次体验产品经理 | 首屏要先让用户想点一条路线，不要先让用户读筛选说明 |
| 关系场景内容策划 | 文案从功能说明改为短情绪句，例如下班后、心动线、可撤退、2站 |
| 社交发现 UI 动效设计师 | 首屏采用电影感情侣城市图、暖光、路线光点、轻微脉冲动效 |
| Android 体验开发工程师 | 不重做架构，先调整 HomeScreen 顺序、卡片信息层级和动效 |
| QA 与设备验证负责人 | 验收首页必须可截图成商店图：有大图、有频道、有路线卡、不是文字墙 |

## 3. V0.9.53 版本范围

本轮只做真实 APK 内可见体验，不做虚构商店图：

1. 增加一张新的恋爱感城市主视觉 `hero_date_invitation.png`。
2. 移动端首屏改为：品牌一句话 -> 情绪主图 -> 内容频道 -> 图片路线卡。
3. 将关系筛选后移到路线卡之后，避免第一眼被筛选按钮占满。
4. 首页主图增加轻微路线光点和呼吸感动效。
5. 路线卡减少长文本：
   - 标签从 3 个减为 2 个。
   - 路线说明压缩为 1 行。
   - 站点预览改成横向小胶囊。
6. 路线图片上增加爱心符号和动态路径点，让卡片更像邀约卡。
7. 审计脚本增加感性首屏守护，防止首页退回纯文字/普通列表。

## 4. 不做事项

- 不接真实 Google 登录。
- 不接支付或 Billing 生产配置。
- 不接 Kimi API。
- 不做未授权平台内容抓取。
- 不展示真实热度、评分、UGC 或官方授权。

## 5. 验收标准

- 首页首屏能看到大图、频道和路线卡，而不是大段文字。
- 心动路线频道仍能看到同城校验和站点预览。
- 首页动效不影响点击路线和进入结果页。
- APK 版本为 `0.9.53 / versionCode 71`。
- 源码审计、Debug/Release 构建、Lint、模拟器冷启动和首页截图通过。
