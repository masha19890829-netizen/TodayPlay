# TodayPlay V0.9.58 全员设计先行会议纪要

日期：2026-06-15

## 会议结论

V0.9.58 不进入功能堆叠，不先继续写代码。本轮先完成“首页想点、卡牌想刷、转场清楚、结果页知道下一步”的设计闭环，再交给 Android 开发落地。

核心目标：

- 首页从文字说明页改成图片化路线推荐页。
- 首屏减少文字，只保留 Hero、频道、首张路线卡和明确 CTA。
- 用瀑布流/单列大卡承载路线推荐，不再把路线解释写满屏。
- Loading 改成票根/路线点逐步生成，不再是普通等待。
- 结果页首屏必须看到地图预览、当前站、下一步行动。
- 折叠屏、窄屏、普通手机都必须先有 QA 验收矩阵。

## 已形成的产品共识

1. 首页不是说明书。
   首页只负责让用户开始第一条路线。隐私、账号、商店、城市包完整列表、多语言设置和数据来源说明全部后置。

2. 新用户前 2 分钟路径必须成立。
   打开 App -> 看到 Hero 和首张路线卡 -> 点 Hero CTA 或卡片开始 -> 票根 loading -> 结果页看到当前站和下一步行动。

3. 卡牌先图片，再字段。
   推荐卡只保留：封面图、路线标题、城市、关系/频道、时长、站点数、一句推荐理由、两个站点预览、开始/收藏/邀请。

4. 地图缩略先做示意，不接 SDK。
   V0.9.58 先用路线点、线、城市标签和当前站高亮表达路线，不引入真实地图 SDK、定位权限或外部地图密钥。

5. 动效要少但关键。
   动效只服务四个瞬间：频道切换、卡片点击、路线生成、打卡完成。不做大面积粒子、复杂 3D、重 Lottie 依赖。

6. 结果页首屏必须行动化。
   结果页不是报告页，首屏要突出地图预览、当前站、下一步行动、进度和主按钮。

## V0.9.58 设计交付拆分

### P0：首页首屏减负

Owner：产品经理 + UI 动效

要求：

- 第一屏最多 3 个主要信息层级：Hero、频道、路线卡。
- Hero 只放一句承诺和一个主 CTA。
- 频道最多保留：今天可做、心动路线、今晚组局、慢慢走。
- 首张路线卡必须可见，按钮不能被底部导航挡住。

完成定义：

- 新用户 5 秒内能回答“这是帮我生成今晚城市路线的 App”。
- 360dp 宽度下主 CTA 可见。

### P0：瀑布流/单列路线卡

Owner：产品策划 + UI 动效 + Android

要求：

- 普通手机先用单列大卡，折叠屏/宽屏可双列。
- 图片占比高，文字少。
- 每张卡保留两个站点预览。
- 主按钮为“开始”，收藏/邀请弱化。

完成定义：

- 首页不像文字墙。
- 首屏至少露出一张完整可点路线卡。

### P0：Loading 票根生成

Owner：UI 动效 + Android

要求：

- 不用普通 spinner。
- 用票根轮廓、路线点、短状态文案表达生成过程。
- 状态文案不超过 8-14 个中文字符。

完成定义：

- 用户能看懂“正在生成路线”。
- 成功后进入结果页，失败有重试/返回。

### P0：结果页路线驾驶舱

Owner：产品经理 + UI 动效 + Android

要求：

- 地图预览靠前。
- 当前站和下一步行动首屏可见。
- 主行动按钮清楚。
- 替代、换点、恢复仍放在站点卡，不挤进驾驶舱。

完成定义：

- 用户不用读长文，就知道第一站去哪、做什么。

### P1：三频道视觉区分

Owner：产品策划 + UI 动效

要求：

- 心动路线：玫瑰粉/酒红/暖金，黄昏、河边、咖啡、灯光街角。
- 今晚组局：暖橙/奶油黄/城市蓝，朋友桌边、夜市、餐桌。
- 慢慢走：雾蓝/薄荷绿/浅米白，公园、书店、海边、人行道。

完成定义：

- 遮住频道标题后，仍能从颜色和图形气质区分频道。

### P1：适配与 QA 矩阵

Owner：设备截图 QA + Android

要求：

- 普通手机、窄屏、折叠屏外屏、折叠屏内屏、横屏都要有验收标准。
- 检查标题溢出、乱码、底部遮挡、图片裁切、按钮可点。

完成定义：

- 每轮实现后必须附对应截图证据。

## 冲突与负责人裁决

1. 真瀑布流 vs 稳定单列
   - UI 希望有瀑布流观感。
   - Android 建议普通手机不做复杂 masonry。
   - 负责人裁决：V0.9.58 普通手机先做单列大卡，宽屏再双列；用图片高度和卡片节奏做“瀑布流感”，不引入复杂布局风险。

2. 地图真实 SDK vs 抽象地图预览
   - 用户希望有地图。
   - 发布和 Android 均提醒真实地图 SDK 有权限、密钥和授权成本。
   - 负责人裁决：本轮做抽象地图预览，不伪装真实地图数据。

3. 商店转化 vs 合规边界
   - UI/产品希望截图更吸引人。
   - 发布/Billing 提醒不能写假热度、假评分、未上线支付。
   - 负责人裁决：截图卖点写真实可运行路径，不展示未上线能力。

## 下一步派工

## 流程问题复盘

本轮暴露的问题：角色线程完成后不会自动把报告推送回负责人线程。

修正后的流程：

1. 负责人派工。
2. 等角色线程完成。
3. 负责人主动读取线程报告。
4. 负责人形成中央会议纪要。
5. 负责人把决策反馈给角色线程。
6. 角色按决策进入下一轮。

从 V0.9.58 开始，每次全员协作必须生成中央纪要，不再只依赖分散线程输出。
