# TodayPlay V0.9.73 Quiet Waterfall Design Blueprint

日期：2026-06-23  
阶段：设计先行，不进入本文件范围内的 Android 开发  
目标：继续解决“字多、不想点、转场存在感弱、卡片过满”的核心体验问题。

## 一句话方向

TodayPlay V0.9.73 要像一本会自己翻页的今日路线杂志：打开先看到可玩的画面和路线卡，点一下卡片才展开细节，生成过程像纸片显影，不再把用户压进说明文和表单。

## 团队会议纪要

| 工种 | 决策 |
| --- | --- |
| 首次体验产品经理 | 首屏只回答“今天有什么值得点”，不解释完整产品。用户先刷卡，再用一句话改。 |
| 应用美术 UI 动效设计师 | 视觉从“票根纸感”升级为“静默电影分镜”：纸、柔光、路线线条、低饱和城市色块。 |
| UX 文案编辑 | 首屏非卡片文案压到 28 个中文字符以内；卡片只留短标题、城市时间、两站预览和一个主动作。 |
| AI 路线逻辑负责人 | 每张卡必须带 routeIntentId，点卡后结果页继承标题、城市、站点和理由。 |
| 设备截图 QA | 320dp 单列，360dp 以上可双列；折叠屏必须限制卡片宽度，不允许底部入口盖住卡片。 |

## 当前 V0.9.72 仍未解决的问题

- 首页已经有瀑布流，但每张卡仍显示过多字段：城市/时间/预算、移动强度、来源状态、站点、三个按钮。
- `自己说一句` 已经不遮挡，但视觉上仍像普通表单入口，没有成为轻量 AI 魔法入口。
- 结果页首屏功能丰富但层级重：票根、适配理由、调参、地图、执行卡连续出现，用户要读太多。
- 转场已有路线动画，但缺少从“我点了这张卡”到“这张卡变成我的路线”的连续感。
- 视觉色板仍偏单一暖纸色，缺少克制的绿色、雾蓝和夜色层次。

## V0.9.73 首屏结构

保留：

- `TodayPlay` 品牌。
- 当前城市胶囊。
- 设置入口。
- 5 个以内场景 chips。
- 视觉路线卡瀑布流。
- 一句 AI 输入入口。

删除或后置：

- 来源状态全文。
- 多按钮并列。
- 大段功能解释。
- 预算、移动强度、合规说明同时挤在卡片内。
- 结果页之外的复杂调参解释。

## 瀑布流卡片信息结构

每张首屏卡最多 6 个可见信息点：

1. 主视觉图或路线线条。
2. 8-10 字短标题。
3. 城市 + 时长。
4. 两个站点预览。
5. 一个情绪/关系标签。
6. 一个主动作：`开始`。

卡片详情页再显示：

- 预算。
- 移动强度。
- 来源状态。
- 第 3 站。
- 保存、邀约、换一幕。

## 首页交互逻辑

```text
冷启动
  -> 开屏票根淡入 900ms
  -> 首页卡片错峰出现
  -> 用户刷卡或点 chip
  -> 点卡进入卡片详情
  -> 点开始进入纸片显影
  -> 结果页显示地图和当前站
```

自由输入逻辑：

```text
点“自己说一句”
  -> 背景卡片变淡
  -> 输入纸片上浮
  -> 用户输入一句需求
  -> AI 生成 3 张候选卡
  -> 用户选一张
  -> 同一张卡进入结果页
```

验收口径：生成后出现 3 张候选路线卡，且三张卡的标题、站点顺序和适配理由必须明显不同。

## 结果页首屏结构

首屏只展示：

- 路线标题。
- 城市 + 时间。
- 迷你地图路线。
- 当前站。
- 主按钮：导航当前站。
- 两个轻操作：更安静 / 更便宜。

下滑后展示：

- 适配理由。
- 三幕脚本。
- 来源状态。
- 保存/分享/邀约。
- 任务、积分、照片挑战。

## 文字密度红线

- 首屏非卡片文字总量不超过 28 个中文字符。
- 卡片标题不超过 10 个中文字符。
- 卡片元信息不超过 1 行。
- 卡片站点预览最多 2 个。
- 卡片首屏只出现 1 个主按钮。
- 任何 320dp、小屏、折叠屏外屏不得出现单字独占一行。

## 视觉方向

关键词：

- 静默电影分镜。
- 纸片显影。
- 城市黄昏。
- 低饱和日系动画感。
- 轻路线线条。
- 留白和空气感。

禁用：

- 模仿具体动画导演或现成作品风格。
- 霓虹科技蓝。
- 大面积紫色渐变。
- 玻璃拟态。
- 卡片内堆标签。
- 三个等权文字按钮挤在同一行。

## 开发前验收

进入 Android 开发前必须具备：

- V0.9.73 设计板。
- 首页、输入、生成、结果四个关键画面。
- 动效时长和状态表。
- 小屏、普通屏、折叠屏的字段裁剪规则。
- 审计脚本能检查 V0.9.73 设计交付存在且无乱码。

## QA 硬门槛

V0.9.73 不允许只验证首页。开发完成后必须同时验证：

- 普通竖屏：首页、loading、结果页。
- 320x640：首页、输入入口、结果页首屏、结果页当前行动卡。
- 折叠屏：首页、结果页。
- 横屏：首页、loading、结果页。
- 结果页宽屏最大内容宽度，避免卡片在折叠屏内屏被拉成超宽 banner。
- 报告和 XML 按 UTF-8 读取，产物不得包含 `�`、`ï¿½`、`����` 等可见乱码标记。
