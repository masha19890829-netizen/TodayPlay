# TodayPlay V0.9.66 时光电影视觉地图设计方案

Date: 2026-06-17
Owner: 项目负责人 / 版本统筹

## 1. 设计方向

V0.9.66 沿用用户认可的示意图方向：电影票根、三幕路线、城市镜头、沉静纸感、地图路线图。

这个功能不是电影资讯频道，也不是影迷百科。它的产品目标是：用户用一句话说出今天想要的电影感，TodayPlay 把城市、关系、预算、体力和情绪翻译成一条可以执行、可以保存的“今日电影票路线”。

## 2. 团队共识

### 首席产品经理

验收标准不是“加了时光电影按钮”，而是用户能完成闭环：

1. 输入一句电影感需求。
2. 选择一张电影票候选卡。
3. 进入三幕结果页。
4. 看懂地图路线图。
5. 完成当前镜头。
6. 愿意保存这张今日电影票。

结果页第一屏必须优先出现“片名式标题、票根、地图路线、当前镜头”，不能先解释算法。

### 应用美术 UI 动效设计师

结果页应像一张“今晚专属电影票”，而不是路线说明书。

核心结构：

- 顶部票根：片名式路线标题、城市、时长、预算、可信状态。
- 地图路线图：低饱和浅色地图、柔和路线线、编号节点。
- 三幕卡：Act 01 / Act 02 / Act 03，每幕包含地点、时间和一句行动提示。
- 保存动作：后续做票根盖章、胶片长图、轻量撕票反馈。

适配原则：

- 小屏压缩票根高度，片名最多 2 行。
- 折叠屏不把卡片拉得过宽，后续应做左票根地图、右三幕路线。
- 横屏后续应改为左地图、右场景，不再用竖屏单列。

### AI 路线逻辑与内容可信负责人

地图和票根可以承载电影感，但不能把 AI 联想包装成真实取景地。

内容状态分四类：

- 已核验取景地。
- 公开线索待核验。
- 电影感灵感。
- 本地样例。

V0.9.66 当前使用“电影感灵感 / 本地样例”口径，不使用电影剧照、海报、片名 Logo、明星肖像或长台词，也不声称官方合作、真实取景或同款机位。

## 3. V0.9.66 已落地 UI

首页：

- quick chip 从“电影感”改为“时光电影”。
- 点击后进入电影感意图识别。

候选卡：

- 支持电影感意图。
- 候选卡中出现电影感路线方向。
- sourceNote 显示“真实取景地关系需来源核验，不代表官方授权”。

结果页：

- 新增 `TimeCinemaTicketCard`。
- 新增 `TimeCinemaRouteMap`。
- 新增 `TimeCinemaSceneRow`。
- 票根显示 `TODAY WAS PLAYED / 今日电影票`。
- 场记板显示“先看地图线路，再按三幕完成今天；来源状态会单独标注。”
- 地图路线图显示城市、路线线条、当前站脉冲点。
- 三幕路线显示 Act 01 / Act 02 / Act 03。
- 底部来源提示说明当前内容为本地样例或待核验电影感地点。

## 4. 后续必须继续推进

V0.9.67 优先方向：

- 结果页横屏和折叠屏保持当前路线状态，不因尺寸切换回开屏动画。
- 候选卡第一张直接变成“时光电影路线”，而不是“电影感打卡局”。
- 增加可保存“今日电影票”分享卡。
- 给每个 Scene 增加短地点名、路线角色和来源状态。
- 建立电影地点来源表，逐步支持“已核验取景地”。

## 5. 可信边界

当前版本仍是本地样例内容。任何“取景地 / 拍摄地 / 名场面 / 同款机位”说法必须等后续来源系统和人工核验完成后才能放开。
