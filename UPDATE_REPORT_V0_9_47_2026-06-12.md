# TodayPlay V0.9.47 更新报告 - 2026-06-12

## 1. 本轮决策

这轮先按项目组内部会的结论收口：TodayPlay 不再按“全球随机景点推荐器”推进，而是重新定位为“熟人关系里的今日相处路线脚本 App”。

完整设计与内容运营方案已输出到：

- `PRODUCT_REPOSITION_AND_CONTENT_OPS_PLAN_2026-06-12.md`

本轮参与角色与分工：

- 首次体验产品经理：重新定义前 2 分钟留存路径。
- 竞品与内容运营研究员：拆解社交发现、本地生活、行程工具的可借鉴点和避雷点。
- 关系场景内容策划：定义路线脚本包、城市包、关系场景标签。
- 社交发现 UI 动效设计师：把首页从文字填表改成图片路线流和城市路线包。
- 路线逻辑开发工程师：落实同城硬约束，阻断跨城拼路线。
- 首次启动体验 QA：验证启动、首页、结果页、路线城市一致性。

## 2. 用户指出的问题

- 首页仍有“全球热门城市 / 全球灵感”入口，用户不知道这是哪里。
- 可执行路线不可信，例如上海武康路不应该和深圳湾放在同一天路线里。
- 首页虽然做了瀑布流，但如果内容池和路线逻辑不可信，体验仍然会崩。
- 后续接 Kimi API 或本地 AI 之前，必须先有硬规则，AI 不能随便编地点或绕过同城校验。

## 3. 本轮已完成改动

- 首页推荐默认城市改为具体城市路线包：
  - 情侣：上海
  - 暧昧中：上海
  - 朋友：广州
  - 家人：深圳
  - 一个人散心：杭州
- 首页顶部标签从“全球灵感”改为“城市路线包”。
- 路线生成器新增 `routeCity` 锁定流程：
  - 先确定本条路线唯一城市。
  - 只从该城市 POI 池中选择站点。
  - 生成结果的城市、成员偏好城市、候选数量全部按同一 `routeCity` 记录。
- QA 回归脚本新增 “Same-city route planning guard”：
  - 阻止首页恢复到“全球热门城市”作为可点击入口。
  - 阻止生成器缺少同城筛选逻辑。

## 6. 下一轮建议

- 内容运营先补第一批城市路线包：上海、深圳、广州、杭州，每城至少 8 条路线脚本包。
- UI 美术继续优化卡片图片风格、路线小地图、完成反馈和分享卡，不再堆长文案。
- 开发下一步做距离/区域约束：90 分钟路线优先同区或近邻区域，半日路线才允许跨区。
- QA 增加自动化路径：每个首页场景至少生成一次，检查结果页城市不混用。
- Kimi API 只作为“文案润色和排序助手”接入，不能自己发明 POI。
