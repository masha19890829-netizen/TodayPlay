# TodayPlay V0.9.58 制作人试玩思考报告

日期：2026-06-15  
身份视角：项目制作人 + 外部测试用户第一视角  
测试版本：`0.9.58 / versionCode 76`  
测试包：`dist/TodayPlay-v0.9.58-external-test.apk`

## 一句话判断

V0.9.58 已经从“路线工具”明显进化成了“有视觉记忆点的路线体验 App”，首页不再是文字墙，路线生成路径能跑通，结果页也能看到地图预览和当前站行动。

但它现在仍然像一个“很有方向的外测原型”，还不是一个“看商店截图就想立刻下载并长期使用”的成熟产品。最大问题不是技术崩溃，而是产品承诺、频道心智、收藏/历史留存、卡片到结果页的一致性还没有完全闭环。

## 本轮试玩路径

本轮重新安装并清空数据后试玩：

1. 冷启动进入首页。
2. 浏览首页 Hero、频道和首张路线卡。
3. 切换 `心动路线`。
4. 点击路线卡 `开始`。
5. 观察 Loading。
6. 进入结果页，查看地图预览、当前站、导航和完成按钮。
7. 下滑结果页，查看执行卡、进度、摘要。
8. 返回首页，进入设置页。
9. 进入历史页。
10. 点击底部 `收藏` 入口，观察实际页面。

试玩截图存放：

```text
dist/producer_playtest_v0_9_58/
```

本轮日志未发现崩溃标记：

```text
FATAL EXCEPTION / AndroidRuntime / ANR / Unable to start activity: 0
```

## 已经做对的地方

### 1. 首页终于有“想点一下”的气质

首页首屏有大图、短承诺、频道和路线卡，已经不像早期满屏文字。用户第一眼能理解这是“帮我安排今天路线”的产品。

好的点：

- Hero 图片建立了情绪。
- `给我一条心动线` 是明确行动。
- 首张路线卡有城市、关系、站点预览、收藏、开始、邀请。
- 同城校验和低压力标签能降低用户担心。

### 2. 结果页变得更可执行

结果页能看到：

- 路线标题
- 地图预览
- 当前站
- `导航当前站`
- `打开地图`
- `完成 +35`
- 站点进度

这比单纯生成一堆文字强很多。用户能知道“现在去哪、下一步做什么”。

### 3. 视觉方向基本成立

浪漫极简、票根、城市黄昏、纸感背景的方向是对的，和最初设计稿的气质已经接近一些。比 V0.9.56 的占位感插画好很多。

### 4. 折叠屏方向是可控的

之前 QA 的折叠屏截图显示：

- 竖屏可双列展示卡片。
- 宽屏可左侧内容流、右侧大 Hero。
- 标题没有明显大面积溢出。

这对用户的 OPPO 折叠屏问题是一个实质进步。

## 主要问题分级

## P0：会影响外测理解或信任的问题

### 1. 底部 `收藏` 入口实际打开的是 `灵感开局`

试玩时点击底部第二项 `收藏`，进入的是：

```text
灵感开局 / QUICK START
```

页面内容是“直接选一条路线”，不是收藏夹。

影响：

- 用户会认为收藏功能是假的。
- 已点击卡片 `收藏` 后不知道去哪找。
- 底部导航心智混乱。

建议：

- 二选一：
  - 如果它是灵感页，底部名称改为 `灵感`。
  - 如果它是收藏页，必须做真正的收藏列表。
- V0.9.59 优先做真实收藏列表，最少要能显示用户在首页点过收藏的路线卡。

### 2. 首页默认过度偏“约会/心动”，弱化了产品的通用性

首页 Hero 是“今晚，别只聊天”，首张卡是“下班 90 分钟心动线”。这很有情绪，但会让非情侣用户第一眼误以为 App 只做约会。

影响：

- 朋友局、独处、家人用户可能不继续探索。
- 商店截图会把产品定位窄化。

建议：

- 首页默认频道 `今天可做` 应该混排三类人群：
  - 一张心动路线
  - 一张朋友局
  - 一张慢慢走
- Hero 文案可以更通用：
  - `今晚去哪，不用再想`
  - `给你一条刚好能出门的路线`
- `心动路线` 可以作为频道强表达，而不是默认唯一气质。

### 3. 卡片标题和结果页标题不完全连续

从 `下班 90 分钟心动线` 进入后，结果页标题显示：

```text
不费力也很可爱的约会局
```

情绪是对的，但用户可能会疑惑：我点的那条路线去哪了？

影响：

- 降低路线卡承诺的可信度。
- 用户会感觉生成结果被“重新洗牌”。

建议：

- 结果页主标题优先沿用路线卡标题。
- 可把生成的新标题放在副标题或“今日脚本名”里。
- 地图站点必须和卡片站点完全一致，除非明确告诉用户“已为你微调”。

### 4. Loading 文案“全球精选样例”容易误导

当前 Loading 出现：

```text
正在整理全球精选样例
```

但本轮内容是本地样例/运营样例，不是全球真实精选。

影响：

- 外测用户可能误以为内容来源更真实、更大规模。
- 以后做合规审查时容易被认为夸大。

建议：

- 改成：
  - `正在整理本地样例路线`
  - `正在排一条低压力路线`
  - `正在把站点串成今天的路线`

### 5. 设置页公开显示“正式上架前配置”

设置页里支持邮箱、隐私政策 URL 都显示：

```text
正式上架前配置
```

影响：

- 给外部测试人员很强的未完成感。
- 真遇到问题时没有反馈入口。

建议：

- 外测包至少提供一个反馈入口：
  - 邮箱
  - 问卷链接
  - 复制反馈模板
- 未配置项不要暴露技术状态，可以显示：
  - `外测反馈通道准备中`
  - 或直接隐藏不可用按钮。

## P1：影响转化和留存的问题

### 6. 历史页像数据报表，不像“回忆/通关卡”

历史页展示关系、路线、地点打卡、积分、时长、完成、隐藏任务、关键词。这些信息完整，但情绪不足。

影响：

- 用户不会因为历史页想回来。
- 没有“今天玩过了”的成就感。

建议：

- 历史页改成 `玩过的今天` 或 `今日回忆卡`。
- 每条历史要有封面图、城市、路线名、完成度、获得积分。
- 完成后生成一张可分享的“票根卡”。

### 7. 完成按钮有潜力，但完成反馈不够强

`完成 +35` 是好的游戏化入口，但完成前后缺少更强反馈。

建议：

- 点击完成后给轻量动效：
  - 点亮当前站
  - 积分跳动
  - 下一站浮现
- 完成第一站后给一句鼓励：
  - `第一站完成，下一站更轻松`
- 完成全路线后进入通关卡。

### 8. `邀请` 按钮还不够像真实分享

首页路线卡有 `邀请`，但用户会期待它能立刻发给同行的人。

建议：

- 如果真实分享未接好，按钮改为 `复制邀请文案` 或 `生成邀请卡`。
- 生成内容要非常短：
  - `今晚走这条？2 站，90 分钟，上海。`

### 9. 频道条可用，但认知还不够稳

频道条横向滚动后会只显示部分频道，用户可能不知道还有 `城市包`。选择频道后频道条位置变化也会带来轻微迷失。

建议：

- 首屏只展示 4 个强频道：
  - 今天可做
  - 心动路线
  - 今晚组局
  - 慢慢走
- 城市包放到首页下方专题区，不挤在频道条里。
- 当前频道下方增加一句短说明，例如：
  - `适合刚开始认真聊天`
  - `适合下班后叫上朋友`
  - `适合一个人慢慢回血`

### 10. 缺少“我在哪个城市”的主动入口

现在路线卡有城市，但用户不能第一时间声明自己所在城市。

影响：

- 外地用户看到上海/广州/杭州会觉得内容离自己远。
- 同城校验虽有，但不是用户自己的同城。

建议：

- 首页增加轻量城市切换，不要回到以前复杂城市包。
- 方式可以很轻：
  - 顶部小字 `当前推荐：上海`
  - 点击后弹出城市选择底 sheet。

## P2：视觉和细节问题

### 11. Hero 副标题在图片上对比度偏弱

Hero 里的灰色副标题压在图片上，部分设备上可能不够清楚。

建议：

- 副标题加更稳定的浅底遮罩。
- 或把副标题移到图片底部白雾区。

### 12. 卡片右上角 `心` 字装饰略显奇怪

路线卡右上角有一个大 `心` 字，可能像未加载的图标，也可能在部分字体环境里不稳定。

建议：

- 换成明确图标或小印章。
- 例如：爱心线稿、票根章、频道色小标。

### 13. Quick Start 页面仍出现 `mock` 字样

`灵感开局` 页面说明里出现：

```text
mock 地点
```

影响：

- 外部测试用户会感觉产品还很工程化。

建议：

- 改成 `本地样例地点` 或 `示例地点`。
- 更好的方式是完全用户化：
  - `每条路线都先用本地样例跑通，正式版会接入更多城市内容。`

### 14. 结果页顶部 Hero 仍占据较多首屏空间

结果页顶部票根图很好看，但外部用户进入结果页时最想知道的是：

- 现在去哪
- 怎么去
- 下一站是什么

建议：

- 结果页首屏进一步压缩 Hero。
- 或让 Hero 右上角承载当前站，不只是情绪图。

## V0.9.59 建议目标

建议 V0.9.59 不继续大扩功能，先做：

```text
V0.9.59 外测反馈闭环版
```

核心目标：

让外部测试用户从打开到生成、收藏、完成第一站、留下反馈，形成完整闭环。

## 外测策略建议

当前 V0.9.58 可以外测，但应该控制测试目标：

适合测试：

- 第一眼是否想点。
- 首页是否比文字墙好。
- 三个频道是否看得懂。
- 路线卡是否可信。
- 结果页是否知道下一步。
- 折叠屏是否还溢出。

不适合测试：

- 真实登录。
- 真实支付。
- 真实地图。
- 大规模城市内容。
- 长期留存。

建议外测问题：

1. 你打开后 5 秒知道这个 App 是干什么的吗？
2. 你最想点哪张卡？为什么？
3. 你觉得它只适合约会，还是朋友/独处也能用？
4. 生成结果后，你知道第一站去哪吗？
5. 你会把路线发给别人吗？
6. 你点了收藏后，知道去哪找吗？
7. 哪些文字看起来像内部测试或未完成？
8. 你的手机上有没有文字挤出、乱码、按钮挡住？

## 制作人结论

V0.9.58 方向正确，值得继续推进，也可以给小范围外部测试人员试用。但下一轮不要急着加 Kimi、支付或更多复杂能力。先把“首页想点、生成可信、收藏找得到、完成有反馈、问题能回传”这 5 个闭环打通。

产品现在最缺的不是更多路线，而是更稳的用户心智：

```text
我打开它，不用想，就能选一条适合今晚的路线；
我点了哪条，结果页就尊重哪条；
我走完一站，有反馈；
我喜欢一条，真的找得回来；
我遇到问题，可以告诉团队。
```

这条闭环跑顺后，TodayPlay 才有资格进入更大范围外测和商店截图打磨。
