# V0.9.35 单点重抽路线编辑更新报告

日期：2026-06-11  
版本：`0.9.35 / versionCode 53`  
负责人视角：项目负责人调配产品经理、市场调研、开发与测试  
本轮主题：保留整条路线，只替换用户不喜欢的某一站

## 背景

V0.9.34 已经允许用户把某一站改成“替代玩法”，解决了路线不必卡死的问题。但产品经理视角仍有一个体验缺口：如果用户不是想做替代动作，而是单纯不喜欢这个地点，仍然需要整局重抽，导致喜欢的站点也被丢掉。

本轮推进“路线编辑”的第一步：做一个不依赖后端、不伪造真实全局搜索的单点重抽。它使用当前本地同城候选 POI，保留路线结构，只替换当前站。

## 已交付

1. 地点卡新增“换一个地点 / Replace this stop”
   - 用户可以在单个地点卡上替换当前站。
   - 替换不影响其它站点、主线任务、隐藏任务和分享路径。
   - 替换成功后会给轻提示。

2. 同城候选重抽算法
   - 从同城候选 POI 中筛掉当前路线已经使用过的地点。
   - 优先选择同类别地点。
   - 其次选择标签更匹配路线兴趣和当前站标签的地点。
   - 再按停留时长接近程度排序。

3. 替换结果真实保存
   - 替换后的路线会写入本地历史记录。
   - 用户离开结果页后再从历史打开，能看到替换后的路线。

4. 清理旧站进度
   - 替换后清掉该站旧的 `Completed / Skipped` 状态。
   - 清掉该站旧反馈。
   - 清掉该站旧奖励积分，避免旧地点的积分污染新地点。

5. 回归审计新增保护
   - `playstore/app_regression_audit.py` 新增 `Route stop replacement path`。
   - 后续预检会检查单点重抽算法、仓库保存链路、ViewModel 接线和 UI 入口仍然存在。

6. 版本与发布材料同步
   - App 版本推进到 `0.9.35 / versionCode 53`。
   - 同步更新发布计划、Play Console 字段清单、官方要求对照、Release Ops 行动计划和项目总控状态。
   - Play 内测说明新增 V0.9.35。

## 下一轮建议

1. 产品经理优先评审“换一个地点”和“改用替代方案”的差异是否足够清楚。
2. 下一轮可做“替换候选预览”：点按钮前展示候选地点名称，而不是直接替换。
3. 市场调研继续拆解路线编辑和城市主题卡产品，找低风险可复用机制。
4. 真机可用后优先验证：先打卡再替换、先替代再替换、历史打开后路线是否保持替换结果、分享卡是否使用替换后的路线。
