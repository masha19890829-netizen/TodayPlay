# TodayPlay V0.9.48 更新报告 - 2026-06-12

## 1. 本轮目标

继续推进首页和内容体验重构。V0.9.47 已经解决“全球热门城市”和跨城路线问题，但首页卡片仍然偏“氛围标题”，用户需要点进去才知道路线是否靠谱。

本轮 V0.9.48 的目标是让首页卡片直接露出路线证据：

- 具体城市
- 同城校验
- 2-3 个可见站点
- 关系场景安全提示

## 2. 角色分工

- 首次体验产品经理：要求卡片在 3 秒内回答“这条路线去哪、适合谁、靠不靠谱”。
- 竞品与内容运营研究员：参考内容流产品的预览逻辑，避免只给标题不给内容。
- 社交发现 UI 动效设计师：把站点预览做进卡片内部，让瀑布流更像可点击内容，而不是表单入口。
- 路线逻辑开发工程师：把首页卡片站点与本地 POI 城市池保持一致。
- QA：检查首页、冷启动、生成路径、同城路线不回退。

## 3. 已完成改动

- 首页瀑布流卡片新增路线证据区：
  - `上海 · 同城校验 · 低压力`
  - `武康路 Citywalk`
  - `外滩黄昏观景线`
- 不同城市/关系卡片会展示对应站点：
  - 上海：武康路、外滩
  - 深圳：深圳湾、华侨城 / 博物馆
  - 广州：永庆坊、沙面岛
  - 杭州：西湖、小河直街
  - 台北：大稻埕、淡水河岸
- 回归审计新增首页站点预览守门：
  - `FeedRouteProof`
  - `feedRouteStops`
  - `feedRouteProof`
  - `同城校验`
- 版本升级到 `0.9.48 / versionCode 66`。

## 6. 下一轮建议

- 把首页两列瀑布流继续细化成“城市包 / 关系包 / 今晚可做”三个内容频道。
- 给每条路线卡补“预计移动压力”：近距离、同区、跨区、半日。
- 增加内容运营的首批路线包文件，让产品经理和内容策划可以独立补内容，不需要每次改 UI 代码。
- 美术继续优化卡片内路线证据区，下一步可做成更像小地图票根的视觉组件。
