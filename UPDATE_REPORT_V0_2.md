# 今天怎么玩 V0.2 更新总结报告

## 体验视角

本轮以两个视角试用产品：

- 产品经理：关注首屏承诺、主路径效率、留存和分享闭环。
- 20 岁热恋中的女生：关注“想不想发给对象”“会不会觉得填表麻烦”“结果有没有恋爱仪式感”。

本地没有连接中的 Android 设备或模拟器，因此本轮以 APK 元数据、源码页面流和可构建验证为主，未做真机点击录屏。

## 发现的问题

1. 首页虽然漂亮，但价值到达太慢  
   用户必须先进入表单并完成多项选择，才能看到真正的“今日副本”。对“今晚不知道怎么玩”的用户来说，开局成本偏高。

2. 缺少热恋用户需要的“立即惊喜”  
   热恋中的用户不一定想认真填表，更像是想点一下就拿到可以发给对象的灵感。原版本缺少一键场景入口。

3. 结果页一次性消费感偏强  
   生成完虽然能重新生成和分享，但没有“这次副本被收好”的感觉，用户缺少回看和收藏动机。

4. 首页和参考图的产品结构还有差距  
   参考图里有历史脚本/票根库的产品感，原版本只有单次生成流程。

5. 输入页状态保存不完整  
   心情多选使用普通 `remember`，旋转或重组时存在丢失选择的风险。

## 本轮已解决

1. 新增“今晚直接开局”
   - 下班后见面，但不想费脑子
   - 有点暧昧，想自然升温
   - 纪念日，但预算不想爆炸

   这三个预设可以从首页直接生成今日副本，不需要先填完整表单。

2. 新增“历史副本”
   - 首页增加“看看历史副本”入口。
   - 每次生成或重新生成都会进入本次会话历史。
   - 历史页用票根卡片展示标题、关系、时长、关键词。
   - 点击历史副本可以重新打开结果页。

3. 结果页新增收藏动作
   - 顶部操作从装饰爱心升级为“收藏”。
   - 收藏会把当前副本加入历史列表。

4. 输入页状态更稳
   - 心情多选从 `remember` 改为 `rememberSaveable`。

5. 版本升级
   - `versionCode`: 2
   - `versionName`: 0.2.0

6. 新增工作流 Skill
   - 路径：`C:\Users\44277\.codex\skills\product-apk-iteration`
   - 用途：以后可复用“产品体验审查 → 代码迭代 → APK 打包 → 更新报告”的流程。
   - 校验：`Skill is valid!`
