## 2. 本轮发现并修复的问题

### P0：Release 签名配置被 BOM 破坏，AAB 曾是未签名产物

现象：
- `:app:signingReport` 初始显示 release 变体 `Config: null`。
- `jarsigner` 初始检查 `app-release.aab` 时显示未签名。
- `assembleRelease` 生成的是 `app-release-unsigned.apk`。

根因：
- `keystore.properties` 文件头存在 UTF-8 BOM：`EF BB BF`。
- Gradle 使用 Java `Properties.load(InputStream)` 读取时没有剥离 BOM，首个键被读成 `???storeFile`，导致 `storeFile` 实际为 `null`。
- Python gate 使用 `utf-8-sig` 读取，反而剥离了 BOM，所以误判 Upload key 已就绪。

已处理：
- 仅移除 `keystore.properties` 的 BOM，未改动密钥内容。
- 复验后 release 变体已绑定 `releaseUpload`。
- 当前 AAB 内已存在 `META-INF/TODAYPLA.SF`、`META-INF/TODAYPLA.RSA`、`META-INF/MANIFEST.MF`。

建议补强：
- `google_play_submission_gate.py` 增加 release signing 检查：验证 `:app:signingReport` release 不为 null，或直接检查 AAB 顶层 `META-INF/*.SF/*.RSA/MANIFEST.MF`。
- keystore 读取逻辑应模拟 Gradle 的 Java Properties 行为，避免 Python 端剥离 BOM 后误判。

## 5. 功能测试判断

从代码路径和自动化审计看，核心功能链路具备可测试状态：
- 首屏可选择语言、关系、快速生成或进入卡片式配置。
- 创建路线流程具备必填项校验、低压力互动开关、备注输入和异步生成。
- Loading 页面跟随真实生成状态，不再依赖假定时器；失败可重试或返回编辑。
- 结果页支持路线概览、POI 卡、外部地图跳转、手动打卡、积分、本地历史、分享卡。
- 隐私页支持本地历史清除；配置后可打开支持邮箱和 HTTPS 隐私政策。
- 商店页在 `BILLING_VERIFY_ENDPOINT` 未配置时禁用购买，避免未验单付费。

需要真机复核的功能：
- 首次启动、语言切换后所有页面文案是否即时一致。
- 手动创建路线的 7 步流程在 360dp 以下是否有按钮遮挡。
- 结果页长地点名、日文/韩文/西文文本是否仍能阅读。
- 外部地图跳转在未安装高德、无浏览器、剪贴板 fallback 时的表现。
- Billing 在 Play 内部测试安装包中的商品查询与未配置/已配置 endpoint 状态。

## 6. 适配与体验问题

P1：商店截图素材需要重采
- 现有 `playstore/screenshots` 截图是 2026-06-08 的草稿，和当前 V0.9.25 不完全一致。
- 截图中结果和打卡页出现 `?` 分隔符，例如 `3 stops ? 2h 30m ? ?100-300`，不适合作为正式商店素材。
- 建议用当前 APK 在真实模拟器/真机重走流程后重新截图。

P1：版本号存在并行漂移
- 用户输入目标是 `0.9.22 / 40`，测试过程中工作区已推进到 `0.9.25 / 43`。
- 建议后续每一轮开始前先锁定目标版本，并在报告、AAB、APK、Play release notes 中保持一致。

P2：提交门禁仍有盲区
- 当前 gate 能检查 keystore 字段是否存在，但之前没发现 release 变体实际未签名。
- 建议新增 AAB 签名检查、release signingReport 检查、截图内容质量检查。

P2：App 层自动化测试为空
- Worker 契约测试已覆盖得不错，但 Android App 内生成器、历史存储、支付客户端、地图 fallback 缺少 JVM 或 instrumentation 测试。
- 建议至少补：路线生成快照测试、历史 upsert/clear 测试、Billing endpoint guard 测试、MapNavigator intent fallback 测试。

P2：依赖升级需要单独回归
- Lint 提示 Compose BOM、Activity、Lifecycle、Billing 均有新版本。
- 当前建议不盲升；等 Android 36 SDK / AGP / Kotlin 工具链升级窗口一起做，并覆盖 UI 与支付回归。
